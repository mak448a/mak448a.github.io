How to compile a Godot project to iOS without a Mac

If you don't have a Mac, there's an easy way to compile Godot projects to iOS.

Date: July 22, 2024

Author: mak448a

Requirements

  • Git installed on your computer
  • A GitHub account
  • Basic Git + GitHub knowledge

Step 1: Setup GitHub repository

First, create a GitHub repository. You may want to set it to private. Pull it to a folder on your computer like so: git pull (github.com/user/yourrepourl)

Step 2: Create local GitHub repository

Download my starter repository here. Next, copy the contents of that folder to your local repository.

Open your local repository in your file manager and open to .github/workflows/build.yml. You may have to show hidden files if you're on Mac or Linux. Replace all occurences of "Example-Project" with your project name.

Export your project using the iOS preset and shove the Xcode Project into your local repository. IMPORTANT! DO NOT INCLUDE SPACES IN YOUR EXPORT FILENAME!!!

Run git commit -m "Upload" and git push.

If you get an error saying that you have exceeded GitHub's upload limit, follow the instructions here.

Step 3: Download your build

Visit your repository URL on GitHub and go to the Actions tab. Click on the topmost entry. If it is still yellow, wait a little longer. If it's green, scroll to the bottom and click the download button. You may have to refresh the page to see the download.
